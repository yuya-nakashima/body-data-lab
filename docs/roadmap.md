# Roadmap

Body Data Lab の将来構想をまとめる。
実装仕様は [features/](./features/) 配下に分離して管理する。

---

## 現在地

- ETL による日次集計（steps_total）が動作している
- メール通知（成功 / エラー）が実装済み
- API / SQLite / ETL の基本構成が稼働中

---

## 近期（フェーズ 1 完成に向けて）

- **日次サマリー通知の充実**
  ETL 完了時の通知を「前日との差分」「週平均との乖離」を含む内容に拡張する
  → 仕様詳細: [features/notifications.md](./features/notifications.md)

- **対象指標の拡充**
  steps_total のみから、sleep / heart_rate / activity_mode を順次追加する

- **データ品質チェック**
  欠損日・異常値の検出をETLパイプラインに組み込む

---

## 中期（フェーズ 2 相当）

- **自動実行基盤**
  現在は手動実行の ETL を cron または EC2 スケジュールで自動化する
  ローカル開発は cron、本番は EC2 + systemd timer を想定

- **群比較・相関分析の追加**
  稽古日 / 非稽古日の差分、睡眠と疲労の相関などをAPIで提供する

- **簡易ダッシュボード**
  取り込みデータの折れ線・週平均・移動平均を可視化する最小UI

---

## 遠期（フェーズ 3 相当）

- **多変量解析によるコンディション合成指標**
  歩数・睡眠・心拍を統合したスコアを PCA / 因子分析ベースで生成する

- **時系列予測・状態推定**
  ARIMA や HMM（隠れマルコフモデル）による短期予測と状態分類

- **ベイズ適応分析**
  蓄積データに基づいた個人特化の事前分布更新

- **収益化構想の実装検討**
  スマートウォッチ連携・アフィリエイト等の収益化方向性の具体化
  → 構想詳細: [features/monetization.md](./features/monetization.md)

- **健康辞書**
  集中力・睡眠・疲労など健康に関する知識を行動とセットで解説するコンテンツ基盤
  通知やデータの「なぜそれをやるのか」を説明できるようにする
  将来的には SEO コンテンツ・アフィリエイト導線として展開する

- **アンケート機能**
  健康・集中力に関する体感データをユーザーから収集し、統計分析と記事コンテンツの強化に活用する
  健康辞書の記事内に選択式1問を埋め込み、回答を蓄積する設計
  → 仕様詳細: [features/survey.md](./features/survey.md)

---

## フェーズ移行の判断基準

| 基準 | 内容 |
|------|------|
| データ蓄積量 | 30日以上の継続取得 |
| 欠損率 | 週7日中5日以上のデータが揃っていること |
| 実装負荷 | 現在の構成で保守可能なレベルにあること |
| 解釈可能性 | 分析結果を自分で説明できること |
