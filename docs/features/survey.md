# アンケート機能

---

## 目的

健康・集中力に関するユーザーの体感データを収集し、
統計的な分析とコンテンツ価値の向上に活用する。

---

## 優先度

**低**（設計のみ、実装は後回し）

理由：
- コア機能（ETL / 通知 / 集中時間）が未完成
- 実装コストに対して即効性は低い
- ただし将来的な価値は非常に高い

---

## コンセプト

辞書 × アンケート × データ分析

```
記事を読む → アンケート回答 → データ蓄積 → 統計表示 → 記事の信頼性向上
```

---

## 機能概要

### ① アンケート表示

- 記事（辞書）内に1問表示
- 選択式（3〜5択）

### ② 回答保存

```
survey_responses
- id
- question_id
- answer
- created_at
```

### ③ 集計表示

例：`ユーザーの62%が「30分以内に集中が切れる」と回答`

### ④ 比較表示（将来）

```
全体: 30分以内 62%
自分: 45分
```

---

## 最初のアンケート案

### Q1: 集中はどれくらい持続するか？

- 15分以内
- 30分以内
- 60分以内
- 1時間以上

---

## 活用方法

| 用途 | 内容 |
|------|------|
| 記事強化 | 結果を記事に反映し信頼性向上 |
| 分析 | 集中時間との相関、行動との関係 |
| 通知との連携（将来） | 「あなたの集中時間は平均より長いです」など |

---

## 設計ポイント

- シンプルに始める（1問のみ）
- 選択式のみ（自由入力は分析困難のため避ける）
- データは蓄積重視

---

## やらないこと（初期）

- 複雑なアンケート・複数質問
- ログイン必須
- 詳細分析UI

---

## 将来拡張

- 複数質問・セグメント分析
- パーソナライズ
- アプリデータとの統合

---

## メモ

アンケートは「統計を実践する場」として重要。
今はコア機能を優先するため、設計のみ残す。
