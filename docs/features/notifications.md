# 通知機能仕様

ETL 完了時に送信する日次サマリー通知の仕様を定義する。

---

## 概要

- **対象**: ETL 実行完了時（`etl/main.py` の `notify_success` 呼び出し）
- **配信方式**: メール（SMTP / `etl/notifier.py`）
- **フォーマット**: プレーンテキスト

---

## 通知に含める情報

| 項目 | 内容 | データ不足時 |
|------|------|------------|
| 対象日 | 集計日（前日） | - |
| 前日の値 | 各指標の当日値 | 「未」と表示 |
| 前日との差分 | 当日値 − 2日前の値 | 「未」と表示 |
| 1週間平均との差 | 当日値 − 過去7日平均 | 「未」と表示 |

データ不足の定義: 当該日のレコードが存在しない、または NULL の場合。

---

## 対象指標

フェーズ 1 では **steps_total** のみ対象とする。

指標追加の優先順位:
1. steps_total（現在実装中）
2. sleep_duration_min（次期対応予定）
3. resting_heart_rate（以降検討）
4. activity_mode（以降検討）

---

## 通知フォーマット（プレーンテキスト）

```
[Body Data Lab] 日次サマリー 2026-03-23

--- 歩数 (steps_total) ---
当日値    : 8,432 歩
前日差分  : +1,204 歩
週平均差  : -568 歩

---
取得日時: 2026-03-24 06:00 JST
```

---

## データ不足時の例

前日データが存在しない場合:

```
[Body Data Lab] 日次サマリー 2026-03-23

--- 歩数 (steps_total) ---
当日値    : 未
前日差分  : 未
週平均差  : 未

---
取得日時: 2026-03-24 06:00 JST
```

週平均に使う7日分が揃っていない場合でも週平均差は「未」とする（部分計算はしない）。

---

## 実装上の注意

- `notify_success` の `message` 引数にフォーマット済み文字列を渡す
- ETL の集計結果（`aggregate_daily_metrics` の戻り値）からサマリーを生成する
- 差分・平均の計算は ETL 内で行い、通知モジュールには完成済み文字列を渡す
- 通知送信の失敗はエラー通知扱いとする（既存の `notify_error` を利用）
