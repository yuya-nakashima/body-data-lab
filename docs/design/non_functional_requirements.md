# Non-Functional Requirements

## 保守性

- 生データ、集計、分析を分離し、後から分析ロジックを差し替えやすくする
- 統計知識と実装仕様をドキュメント参照で結び、属人化を減らす

## 可読性

- データ項目名、単位、分析対象期間を明確にする
- ドキュメントと実装の命名差異を小さく保つ

## 将来拡張性

- 新しい指標を追加しても、既存集計や UI を壊しにくい構造にする
- 上級分析をジョブ化または別レイヤー化できるようにする

## データの正確性

- 重複排除、欠測把握、ソース追跡を行う
- 時刻やタイムゾーンの扱いを統一する
- 再計算可能性を残す

## パフォーマンスの考慮

- MVP では日次集計中心にして計算量を抑える
- 高度分析はオンデマンドまたは非同期化を検討する

## プライバシー配慮

- 身体データは個人性が高いため、保存範囲と送信範囲を最小化する
- サーバー連携時は認証、暗号化、保持方針の整理が必要

## 統計的な非機能観点

- 再現性: 同じ期間・同じ前処理なら同じ結果が再生成できること
- 解釈可能性: 分析結果の前提が追えること
- 誤用防止: 小標本や欠測過多のときに警告できること
